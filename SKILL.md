# 中学教材サイト生成スキル

## ワークフロー

### Phase 1: 要件確認

ユーザーから以下を確認する。不足があれば**最小限の質問**で埋める。

| 項目 | 必須 | 例 |
|------|------|----|
| 教科 | 必須 | 歴史、数学、理科、英語、国語、地理 |
| 学年 | 必須 | 中1 / 中2 / 中3 |
| 単元名 | 必須 | 鎌倉幕府の成立 |
| ページ数 | 必須 | 3〜5 |
| 難易度 | 必須 | 基本 / 標準 / 発展 |
| 技術スタック | 必須 | 静的HTML or Next.js（ユーザーに選ばせる） |
| 授業資料 | 任意 | 図表、画像パス |
| 問題形式 | 任意 | 選択/穴埋め/混合（デフォルト: 混合） |
| 禁止事項 | 任意 | 特定の表現・語彙 |

**曖昧な場合の対応:**
- 単元が広すぎる → ページ分割案を提示して選ばせる
- 学年が不明 → 「中学何年生向けですか？」と1回だけ聞く
- 教科だけで単元不明 → 代表的な単元を3つ提案

### Phase 2: サイトマップ生成

確認した情報をもとにサイト構成案を出力する。

```
例: 歴史・中1「鎌倉幕府」3ページ構成
  index.html            … 単元一覧トップ
  kamakura-intro.html   … 導入＋要点まとめ
  kamakura-quiz.html    … 例題＋練習問題＋解説
```

Next.js の場合は `app/[unit]/page.tsx` 形式のルーティング構成を提示する。

### Phase 3: ページ生成

#### 静的HTML の場合
1. `assets/static-template/` 内のテンプレートをプロジェクトディレクトリにコピー
2. `styles.css` と `quiz.js` はそのまま使用
3. `unit-template.html` の `{{PLACEHOLDER}}` を実際の教材コンテンツに置換して各ページを生成
4. `index-template.html` からトップページを生成

#### Next.js の場合
1. Next.js プロジェクトを初期化（`npx create-next-app@latest`）
2. 共通コンポーネントを作成: Header, SideNav, Breadcrumb, StepIndicator, SectionCard, IntroBox, KeyPoints, ExampleBlock, ChoiceQuiz, FillBlankQuiz, Notice
3. `globals.css` に `assets/static-template/styles.css` のスタイルを適用
4. `app/[unit]/page.tsx` 形式で各単元ページを作成

### Phase 4: コンテンツ作成

教科・学年に応じた表現レベルで本文を作成する。
詳細は [references/content-guidelines.md](references/content-guidelines.md) を参照。

各ページは以下の5セクション構成:

1. **導入** — 単元のテーマを身近な問いかけから始める
2. **要点まとめ** — 箇条書き、太字、色分けで整理
3. **例題** — 1〜2問。解説は折りたたみで表示
4. **練習問題** — 選択式・穴埋め式を混合。`quiz.js` で自動判定
5. **解説** — 各問の正答理由と間違いやすいポイント

**表現の注意:**
- 学年に合わせた語彙を使う（中1→やさしい日本語、中3→入試水準OK）
- 難読漢字にはルビを振る（`<ruby>` タグ使用）
- 一文は短く。中1は40字、中2は60字、中3は80字を目安
- 注意喚起は `.notice--tip` / `.notice--warn` / `.notice--danger` クラスで明示

### Phase 5: 出力確認

以下を出力してユーザーに確認を取る:

1. **サイトマップ**（ファイル構成図）
2. **生成ファイル一覧**（パスとファイルサイズ目安）
3. **各ページのプレビュー概要**（セクション構成と差し替えポイント）

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| 単元名が曖昧 | 候補を3つ提示、1回の質問で確定 |
| 内容量が1ページに収まらない | 分割案を表形式で提示 |
| 学年に対して表現が難しい | 言い換え案を `変更前 → 変更後` で提示 |
| ユーザー指定の問題数が多すぎる | 1ページ5問以内を推奨し、超過分は別ページ提案 |

## やらないこと

- 個人情報を収集するフォームの作成
- ログイン・認証機能の実装
- 外部広告スクリプトの埋め込み
- 政治的・宗教的に偏った表現

## 安全性に関する推奨事項

教材コンテンツは正確性・安全性が特に重要なため、以下を推奨する。

**手動確認の推奨:**
生成された教材は、教育的正確性の観点から公開前に必ず人間が確認すべきである。
特に以下の場合はユーザーに手動確認を促す:
- 歴史的事実や年号を含むコンテンツ
- 科学的な法則・公式を含むコンテンツ
- 練習問題の正答設定

理由: AIが生成した教育コンテンツには事実誤認のリスクがある。
中学生が誤った知識を学習するリスクを最小化するため、
**必ず教科の知識を持つ人間がレビューしてから公開すること**をユーザーに伝える。
